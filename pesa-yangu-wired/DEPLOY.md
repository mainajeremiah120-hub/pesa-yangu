## Step 2 — Deploy Backend on Render

### Option A — Blueprint (automatic, recommended)
1. Go to https://render.com → **New → Blueprint**
2. Connect your GitHub repo
3. Render reads `render.yaml` and creates:
   - **pesa-yangu-api** (Express web service)
   - **pesa-yangu-db** (PostgreSQL database)
   - **pesa-yangu-migrate** (runs migrations on deploy)

4. In the Render dashboard, set these environment variables manually
   (ones marked `sync: false` in render.yaml):
   - `ANTHROPIC_API_KEY` = your key from console.anthropic.com
   - `CORS_ORIGINS` = `https://pesa-yangu.vercel.app,https://pesayangu.app`

5. Trigger a deploy. Migrations run automatically.

6. Copy your API URL — looks like: `https://pesa-yangu-api.onrender.com`

### Option B — Manual setup
1. New → PostgreSQL → name it `pesa-yangu-db` → Free tier
2. New → Web Service → connect repo → Root Dir: `backend`
   - Build: `npm install`
   - Start: `npm start`
   - Add all env vars from `backend/.env.example`
   - `DATABASE_URL` = paste the **External Database URL** from the PostgreSQL service
3. After first deploy, open the Shell tab and run: `npm run migrate`

## Step 3 — Deploy Frontend on Vercel

1. Go to https://vercel.com → **Add New Project**
2. Import your GitHub repo
3. Set **Root Directory** to `frontend`
4. Framework: **Vite** (auto-detected)
5. Add environment variable:
   ```
   VITE_API_URL = https://pesa-yangu-api.onrender.com/api/v1
   ```
6. Click **Deploy**

Your app is live at: `https://pesa-yangu.vercel.app`

---

## Step 4 — Connect Custom Domain

### On Vercel (frontend → pesayangu.app)
1. Vercel → your project → **Settings → Domains**
2. Add `pesayangu.app` and `www.pesayangu.app`
3. At your domain registrar (GoDaddy, Namecheap, etc.), add:
   ```
   Type    Name    Value
   A       @       76.76.21.21
   CNAME   www     cname.vercel-dns.com
   ```
4. SSL is automatic — propagates within minutes.

### On Render (API → api.pesayangu.app) — optional
1. Render → pesa-yangu-api → **Settings → Custom Domain**
2. Add `api.pesayangu.app`
3. At registrar:
   ```
   Type    Name    Value
   CNAME   api     pesa-yangu-api.onrender.com
   ```
4. Update `VITE_API_URL` in Vercel to `https://api.pesayangu.app/api/v1`
5. Update `CORS_ORIGINS` in Render to include `https://pesayangu.app`

## Step 5 — Verify it's working

Open your Vercel URL and:
- [ ] Register a new account → you should be logged in
- [ ] Dashboard loads with empty state (no errors)
- [ ] Add a wallet → appears immediately
- [ ] Add a transaction → appears in Records tab
- [ ] Check Render logs — no errors

## Environment Variables Summary

### Backend (set in Render dashboard)
| Variable              | Where to get it                                 |
|-----------------------|-------------------------------------------------|
| `DATABASE_URL`        | Render → pesa-yangu-db → External URL           |
| `JWT_SECRET`          | Auto-generated by Render (or run `node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"`) |
| `ANTHROPIC_API_KEY`   | console.anthropic.com → API Keys                |
| `CORS_ORIGINS`        | Your Vercel URL + custom domain                 |

### Frontend (set in Vercel dashboard)
| Variable        | Value                                                  |
|-----------------|--------------------------------------------------------|
| `VITE_API_URL`  | `https://pesa-yangu-api.onrender.com/api/v1`           |

## Render free tier note

Render's free web services **spin down after 15 minutes of inactivity**
and take ~30 seconds to wake up on the next request. This causes a slow
first load. To avoid this, upgrade to the **Starter plan ($7/mo)** which
keeps the service always on. The database free tier has no spin-down.
